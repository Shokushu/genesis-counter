Commands

      Commands are Various actions that you can set to your Buttons. This is like a small programming language to generate very specifict effect on your stream.

      {Command name} [Parameters1] [Parameters2+] ... {Millisecond Delay}

      To understand how these commands behave you just need to look at the Order they appear in and the Millisecond delay.

      It will go down the whole list and check in order which one has past its time limit and execute them once the delay has passed.

      The order in which they come up is only relevant if they have the same delay

      If your first command has 3000ms delay and the second one has 0ms, The second one will happen right away and the first one will happen after 3 seconds.

      Each command will only happen once unless re-enable or repeat is used.

      You can insert variable you created anywhere by doing /$variable_name$/

      If the parametter name has a * in the name, that mean its an adaptive box. Meaning...

      If you type numbers, it will take them as they are.

      If you type letters, it will regard it as a variable

      If you start the whole box with ' or " and end it with ' or ". It will regard it as text and remove the '/"(must start and end with the same one)

      If it is something like [x]letters it will regard it as a stack and look at the stack number and name.

    Statements

        If statement

        An if statement is a programming language execution where you compare 2 things and make a decision based on that.

        for exemple, if Variable_name == 2, then it will perform what comes after it if the comparison is true. If not then it won't.

        Couple with an if statement is the "Else" section which will only happen if the if statement is false.

        Lets take our previous exemple. if Variable_name == 2

        Let's say Variable_name contains the number 2, then it will perform whats in the if statement and ignore what's in the else.

        If Variable_name is 3 or anything else but 2. then it will ignore what's in the if statement and perform what's in the else instead.

        Field: Var./real/"string"

          This box can take entire a direct number(a real), some text(a string),a stack value or a variable name(variable can contain either).
          To enter a real just simply put numbers. such as 0 or 100 or -0.0123.
          To enter a string just put whatever text you want and put it between quotation mark, either " or ' so "here is a string".
          To enter a stack value put the stack name with [x] before, so [3]stackname to get value number 3 from stack named stackname, [1+variable*10]stackname is allowed.
          For a variable just put anything else that is letters or _ . So a variable name must start with a letter or _ but may contain numbers as long as it does not start with numbers.

          This is the type of operation you want to perform on the 2 things you want to compare.
          == if they're equal.
          != if they're not equal.
          > if the first box is higher than the second box(does not work on 2 strings)
          < if the first box is smaller than the second box(does not work on 2 strings)
          >= if the first box is bigger or equal to the second box(does not work on 2 strings)
          <= if the first box is smaller or equal to the second box(does not work on 2 strings)

        This will re-enable all the command inside of it if they already happened.

        If the statement is true, the re-enable will still be active unless it as accomplished its amount of time to be performed.

        You can also combine And/Or statement to this same way as you do for If statement.

        you can have repeat inside repeats.

    OBS Source Commands

      Commands sent throught OBSwebsocket by palakis.

      Get the OBSWebsocket plugin to support these commands.

        Scene Switch

        Switch to the selected scene with normal transition.

        Field: scene-name

          Name of the scene

        Change Scene Transition

        Temporarily change the transition of a scene so next time you switch it will be the given transition.

        This should only happen once.

        Field: scene-name

          Name of the scene you are going to switch to

        Field: transitionName

          Name of the transition to use

        Field: transitionDuration

          Duration of the transition in millisecond

        Remove Scene Transition

        remove the temporarily added transition

        Field: scene-name

          Name of the scene that had a temporary transitiona added to it.

        Set Transition Settings

        Set settings on a transition

        Field: transitionName

          Name of the transition to modify.

        Field: transitionSettings

          Settings, This is an object so include which ever value you need here.

        Release Transition Bar

        Release the Transition Bar finishing the transition.

        Set Transition Bar Position

        Set the Transition Bar position.

        Field: position

          Position. must be between 0.0 and 1.0

        Field: release

          Release True or False.

        Source Mute

        Mute the sound of a source.

        Field: source

          Name of the source

        Field: mute

          Select if true/on or false/off.

        Source Toggle Mute

        Toggle on and off the sound of a source.

        Field: source

          Name of the source

        Change Audio Monitor Type

        Change the monitoring type of an andio source, that's the option you see when you right click audio and go in advance.

        Field: sourceName

          Name of the audio source

        Field: monitorType

          type of monitoring to use.

        Source Change Visibility

        Change the visibility of a source.

        Field: scene-name (Optionnal)

          Which scene the source is in. Leave blank for current active scene.

        Field: item

          Name of the source to change visibility.

        Field: visible

          Visibility type, True,False or Toggle.

        Source Change Position

        Change the position of a source.

        Field: scene-name (Optionnal)

          Which scene the source is in. Leave blank for current active scene.

        Field: item

          Name of the source to change position.

        Field: X (Optionnal)

          X position to set the source at. Can leave blank to not modify it.

        Field: Y (Optionnal)

          Y position to set the source at. Can leave blank to not modify it.

        Source Change Volume

        Change the Volume of a source.

        Field: source

          Which scene the source is in. Leave blank for current active scene.

        Field: Volume %

          Percentage to set the volume at, between 0 and 100.

        Source Change Scale

        Change the size of a source.

        The way it's resized is based on what the source's point of origin is set at in OBS.

        Field: scene-name (Optionnal)

          Which scene the source is in. Leave blank for current active scene.

        Field: item

          Name of the source to change size.

        Field: X Scale (Optionnal)

          X Scaling to set the source at. Can leave blank to not modify it. 1.0=100%

        Field: Y Scale (Optionnal)

          Y Scaling to set the source at. Can leave blank to not modify it. 1.0=100%

        Source Change Rotation

        Change the rotation of a source.

        The center point of the rotation is based on the "Positional alignment" found in the edit transform window in obs.

        Field: scene-name (Optionnal)

          Which scene the source is in. Leave blank for current active scene.

        Field: item

          Name of the source to change rotation.

        Field: rotation

          Rotation angle to change the source. 0=normal 180=turned around 360=normal

        Field: sourceSettings

        Field: Fi.Y

        Source Motion: Position Curve

        Move a Source from point A to point B over a given duration AND curves it in a circle pattrern.

        Field: scene

          Which scene the source is in.

        Field: source

        Field: Start Y

        Field: Final Y

          Final Y position.

        Fetch OBS Data

        Allows you to get data from obs and save that in a Variable.

        You have to send an obswebsocket Request command. such has GetSceneItemProperties,GetStats,GetSourceSettings and many others

        See OBSwebsocket protocol for this.

        Exemple: {{"request-type":"GetStats"},fps,value_name,true} will set value_name to the FPS of your stream

        Please keep in mind that since this is over a tcp connection, the value won't be saved instantly.

        you will have to wait a bit before you can see the content of the Variable. So execute this in advance before you start doing your things.

        in general you can expect this to reply on the next frame, but nothing is garanteed even over localhost. so 50ms in advance should be more than enough.

          Naturaly you'd try to search for settings.Filter.Transform.Rotation.X and lioranboard would try to look for those.
          but they dont exist, what you can do instead if this kind of thing happen is
          settings.(Filter.Transform.Rotation.X)
          If lioranboard sees ( after a dot, it will ignore all dots until ). is found and remove the () from both sides
